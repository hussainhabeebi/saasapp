Notifications
Sent via your connected WhatsApp Business number straight through Meta's Graph API — no Chatwoot or n8n dependency. Connect WhatsApp under Settings → Channels first.
How it works:
• Shopify order created → sends the Order Received template
• Order marked paid → sends the Payment Received template
• Fulfillment created (shipping label/tracking added) → sends the Order Shipped template
• Carrier reports delivered → sends the Order Delivered template (best-effort — only carriers Shopify tracks natively report this)
• Checkout abandoned for 60+ minutes with no order → sends the Abandoned Cart template (checked every 20 min)

Don't have a template yet for one of these? Order Received, Payment Received and Order Shipped each have a ⚡ Use Meta Library Template button — creates one of Meta's own pre-approved templates, no review wait, but you'll need to map its variables below afterward. Every event also has a ✨ Create Suggested Template button that submits custom wording instead — usually approved within a few hours, and its variables are already mapped for you.

WhatsApp Templates (Meta)
Sent via your connected WhatsApp Business number when an order's status changes — straight to Meta's Graph API, no Chatwoot dependency. Connect WhatsApp under Settings → Channels first.
How it works:
Changing an order's status here (or via NocoDB) fires a webhook on the ecom_orders table → after.update → POST to:
https://n8n.aiingo.com/webhook/leadvyne-ecom-order-event

• Status → received / confirmed / processing → sends the "Order Received" template
• Status → shipped / dispatched / out_for_delivery → sends the "Order Shipped" template
• Status → delivered / completed → sends the "Order Delivered" template

Note: this only fires on a status change — creating a new order already at that status won't trigger it.
